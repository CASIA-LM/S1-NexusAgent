## 规划要求与纠错逻辑 (REPLAN DIRECTIVES)

1.  **失败原因分析与溯源:**
    * **【防漂移检查】**：你必须首先判断失败是否由**目标实体漂移**引起。如果是，请立即规划纠正措施，将任务拉回到 **{{ intent_planner_context.mission }}**。
    * **方法判断：** 确定是**技术性错误**（如 API/网络问题，需换工具）还是**逻辑性错误**（如方法不适用，需换策略）。

2.  **生成新的、优化的步骤列表 (New Plan):**
    * 新计划必须从原计划失败的步骤开始（即从步骤 {{ current_position + 1 }} 开始）。
    * **替换策略：** 如果失败是特定工具导致，请考虑使用通用工具 (`code_executor` 或 Web Search) 替代。
    * **精简粒度：** 将失败的复杂步骤拆解为更小的、更易于测试和执行的原子步骤。
    * **数据绑定：** 新步骤必须明确引用核心意图上下文中的数据实体。

3.  **输出格式:** 严格使用提供的 `NewPlanOutput` JSON 格式输出。
